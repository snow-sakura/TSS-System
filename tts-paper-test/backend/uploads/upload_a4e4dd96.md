| 文档版本 | v1.0 |

---

## 1. 系统概述

TSS 售票系统是一个基于 Web 的票务管理平台，提供用户注册登录、在线购票、订单管理等功能。系统支持两种用户角色：管理员和普通用户，具有完整的后台管理功能。

### 1.1 系统目标

- 实现在线票务购买流程
- 提供用户管理功能
- 管理目的地和票务信息
- 订单数据统计与展示

---

## 2. 用户角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 管理员 (admin) | 系统管理员 | 全部功能：用户管理、目的地管理、订单管理、数据统计 |
| 普通用户 (user) | 系统用户 | 购票功能、查看个人订单 |

## 3. 页面功能规格

### 3.1 登录页面

**路由**: `/login`

**功能描述**: 用户身份认证入口页面

**页面元素**:
| 元素 | 类型 | 说明 |
|------|------|------|
| 用户名输入框 | text input | 必填，用于输入用户名 |
| 密码输入框 | password input | 必填，用于输入密码 |
| 登录按钮 | submit button | 提交登录表单 |
| 注册链接 | hyperlink | 跳转到注册页面 |

**业务逻辑**:
1. 验证用户名和密码是否正确
2. 密码使用 MD5 加密存储
3. 登录成功后跳转到仪表盘页面
4. 登录失败显示错误提示

**默认账号**:
- 用户名: `admin`
- 密码: `admin123`

### 3.2 注册页面

**功能描述**: 新用户注册页面

**页面元素**:
| 元素 | 类型 | 说明 |
|------|------|------|
| 用户名输入框 | text input | 必填，唯一性校验 |
| 邮箱输入框 | email input | 选填 |

| 确认密码输入框 | password input | 必填，需与密码一致 |

**业务逻辑**:

2. 校验用户名唯一性

### 3.4 购票流程

#### 3.4.1 首页

**路由**: `/`

**功能描述**: 购票入口页面

**页面元素**:
| 元素 | 类型 | 说明 |

**购票流程**:
1. 选择目的地

3. 支付
4. 取票

#### 3.4.2 选择目的地页面

**功能描述**: 用户选择购票目的地

**页面元素**:
| 元素 | 类型 | 说明 |
|------|------|------|
| 目的地下拉框 | select | 显示所有可用目的地 |
| 下一步按钮 | submit button | 进入选项选择 |
| 返回首页按钮 | link | 返回首页 |

**下拉框选项格式**: `代码 - 名称 (¥价格 - 剩余: 库存)`

**业务逻辑**:
1. 仅显示库存 > 0 的目的地
2. 验证目的地有效性和库存

#### 3.4.3 选择票务选项页面

**功能描述**: 用户选择票种和座位类型

| 座位类型下拉框 | select | 普通座 / 舒适座 |
| 下一步按钮 | submit button | 进入支付页面 |
| 返回按钮 | link | 返回选择目的地 |

#### 3.4.4 支付页面

**路由**: `/payment`

**功能描述**: 订单确认与支付

**页面元素**:
| 元素 | 类型 | 说明 |
|------|------|------|
| 订单摘要 | 信息卡片 | 目的地、票种、座位、应付金额 |
| 支付方式下拉框 | select | MCard / 现金 |
| 确认支付按钮 | submit button | 完成支付 |
| 返回修改按钮 | link | 返回修改选项 |

**价格计算规则**:
| 票种 | 座位类型 | 价格倍率 |
|------|----------|----------|
| 单程票 | 普通座 | 1.0x |
| 单程票 | 舒适座 | 1.2x |
| 多次往返票 | 普通座 | 1.5x |
| 多次往返票 | 舒适座 | 1.8x |

**计算公式**: `实际价格 = 基础价格 × 票种倍率 × 座位倍率`

## 4. 数据库设计

### 4.1 用户表 (users)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 用户 ID |
| username | TEXT | NOT NULL, UNIQUE | 用户名 |
| password | TEXT | NOT NULL | 密码（MD5） |
| email | TEXT | - | 邮箱 |
| role | TEXT | DEFAULT 'user' | 角色（admin/user） |
| status | INTEGER | DEFAULT 1 | 状态（1 启用/0 禁用） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 4.2 目的地表 (destinations)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| code | TEXT | PRIMARY KEY | 目的地代码（3位） |
| name | TEXT | - | 目的地名称 |
| price | REAL | - | 基础价格 |
| stock | INTEGER | - | 库存数量 |

### 4.3 订单表 (tickets)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 订单 ID |
| destination_code | TEXT | FOREIGN KEY | 目的地代码 |
| ticket_type | TEXT | - | 票种 |
| seat_type | TEXT | - | 座位类型 |
| price | REAL | - | 实际票价 |
| purchase_time | DATETIME | - | 购票时间 |
| payment_method | TEXT | - | 支付方式 |
| status | TEXT | - | 订单状态 |
| user_id | INTEGER | FOREIGN KEY | 用户 ID |

## 5. 业务规则

### 5.1 购票规则

1. 用户必须登录后才能购票
2. 目的地库存为 0 时不可购买
3. 购票成功后自动扣减库存
4. 删除订单后自动恢复库存

### 5.2 用户管理规则

1. 仅管理员可进行用户管理
2. 管理员账号（admin）不可删除
3. 新注册用户默认角色为普通用户

### 5.3 目的地管理规则

1. 仅管理员可进行目的地管理
2. 目的地代码唯一，不可重复
3. 编辑时代码不可修改

## 6. 接口说明

系统同时提供 Web 页面和 RESTful API 两种访问方式。

### 6.1 API 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/login` | POST | 用户登录 |
| `/api/register` | POST | 用户注册 |
| `/api/users` | GET | 获取用户列表 |
| `/api/users/<id>` | GET/PUT/DELETE | 用户详情/更新/删除 |
| `/api/destinations` | GET/POST | 获取/添加目的地 |
| `/api/destinations/<code>` | GET/PUT/DELETE | 目的地详情/更新/删除 |
| `/api/purchase/price` | POST | 计算票价 |
| `/api/purchase/buy` | POST | 购买票务 |
| `/api/tickets` | GET | 获取订单列表 |
| `/api/tickets/<id>` | GET/DELETE | 订单详情/删除 |
| `/api/dashboard` | GET | 获取统计数据 |

### 6.2 统一响应格式

**成功响应**:
```json
{
  "success": true,
  "message": "操作描述",
  "data": {}
}
```

**失败响应**:
```json
{
  "success": false,
  "message": "错误描述"
}
```
