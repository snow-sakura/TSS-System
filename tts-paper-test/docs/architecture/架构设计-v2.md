## 第一部分：技术选型决策

### 1.1 前端框架选项（请确认）

| 方案 | 技术栈 | 优势 | 劣势 | 适用场景 |
|------|--------|------|------|---------|
| **A: shadcn-vue (推荐)** | Vue 3 + shadcn-vue + Tailwind CSS + TypeScript | 组件拷贝到项目中可完全自定义、暗黑模式原生支持、CSS变量主题化、零运行时依赖、bundle极小 | 需手动安装组件、生态较新 | 追求个性化设计、需要完全控制UI的团队 |
| **B: Vuetify 3** | Vue 3 + Vuetify 3 + TypeScript | 组件最丰富（80+）、Material Design 3、成熟稳定、社区庞大 | 定制难度高、bundle较大、样式覆盖复杂 | 需要快速出活、标准化设计 |
| **C: Element Plus** | Vue 3 + Element Plus + TypeScript | 中文生态完善、企业级组件、表格/表单强大 | 设计风格偏后端、暗黑模式支持较晚 | 企业后台管理系统 |

### 1.2 多智能体框架对比（含成本分析）

| 维度 | CrewAI | LangGraph | AutoGen |
|------|--------|-----------|---------|
| **架构模式** | 角色+任务驱动（Role→Goal→Task） | 状态图驱动（State Graph） | 对话驱动（Conversation） |
| **编排方式** | 顺序/层级/协作流程 | 有向图+条件路由 | 群组对话+轮次管理 |
| **学习曲线** | ⭐ 低（最快上手） | ⭐⭐⭐ 中高 | ⭐⭐ 中 |
| **任务成功率(复杂)** | 71% | **76%** | 68% |
| **每次调用LLM次数** | 中等（4-8次） | **最少（3-5次）** | 最多（15-22次） |
| **相对Token成本** | 1.5x - 2x | **1x (基准)** | 5x - 6x |
| **MCP协议支持** | ✅ 原生支持 | ✅ LangChain生态 | ❌ 需额外集成 |
| **可观测性** | AMP平台($99/月) | LangSmith($39+/月) + 开源 | AgentOps + Azure |
| **开源协议** | MIT | MIT | MIT + CC BY-4.0 |
| **GitHub Stars** | 20k+ | 90k+ (LangChain) | 30k+ |
| **适合场景** | 快速原型、角色明确的任务 | 生产级复杂工作流 | 研究、辩论式推理 |

**成本估算示例**（以GPT-4o mini为例，每次任务）：
- LangGraph: ~5000 tokens × $0.15/1M = **$0.00075/次**
- CrewAI: ~8000 tokens × $0.15/1M = **$0.0012/次**
- AutoGen: ~30000 tokens × $0.15/1M = **$0.0045/次**

**推荐方案**: **CrewAI + LangGraph 混合方案**
- 开发阶段用 **CrewAI** 快速搭建原型（角色定义清晰、开发效率高）
- 生产阶段核心流程迁移到 **LangGraph**（状态控制精准、成本最优）
- 两者可共存，CrewAI v1.15+已支持LangChain工具，生态互通

### 5.2 成本优化策略

| 策略 | 说明 | 预计节省 |
|------|------|---------|
| **模型分层** | 简单任务用GPT-4o-mini/Claude Haiku，复杂任务用GPT-4o/Claude Sonnet | 60-80% |
| **缓存机制** | 相同输入不重复调用，缓存相似度匹配结果 | 20-30% |
| **批量生成** | 一次性生成多条用例，而非逐条生成 | 40% |
| **流式输出** | AI生成流式返回，用户可即时查看编辑 | 体验提升 |
| **本地模型** | 敏感数据/简单任务使用本地部署模型（Ollama） | 100% (无API费用) |
| **Token预算** | 每个Agent设置max_token限制，防止 runaway | 成本可控 |

### 5.3 从CrewAI到LangGraph迁移路径

```
Phase 1 (开发期): CrewAI 快速原型
  Crew(
    agents=[Planner, Generator, Reviewer],
    tasks=[plan, generate, review],
    process=Process.sequential
  )

Phase 2 (生产期): LangGraph 精细控制
  graph = StateGraph(AgentState)
  graph.add_node("planner", planner_node)
  graph.add_node("generator", generator_node)
  graph.add_node("reviewer", reviewer_node)
  graph.add_conditional_edges(
    "reviewer",
    lambda s: "generator" if s.needs_revision else END
  )

好处:
  - 状态可持久化/恢复
  - 条件路由（失败重试/跳过）
  - 支持并行执行
  - Token成本降低40%+
```
