> 版本: v3.0 | 日期: 2026-07-18
> 基于多智能体AI Agent的软件测试全生命周期管理平台

---

## 一、设计理念

### 1.1 核心原则
- **AI First**：所有功能优先由AI Agent驱动实现，人工审核确认
- **Human-in-the-Loop**：AI生成内容均为 Draft，需审核 Approved 后方可入库
- **全链路可追踪**：前端操作日志 + 后端实时日志 + 文件持久化日志
- **模块化可插拔**：三大分类、22个子模块，独立扩展

### 1.2 产品定位
基于大模型多智能体（Multi-Agent）协作的 AI 软件测试平台，覆盖：
- **测试基础**：需求分析 → 测试点提取 → 测试方案 → 测试用例 → 执行管理 → 缺陷管理 → 报告分析
- **基础配置**：环境/大模型/提示词/去AI味/MCP/Skills/Hermes/日志
- **个人中心**：用户/角色/个人设置

## 五、AI Agent 系统设计

### 5.1 Agent 架构

```
用户请求
   │
   ▼
┌──────────────────────────────────────────────────────┐
│                  CrewAI Flow Orchestrator              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │Requirement│ │   Test   │ │   Test   │ │  Report  │ │
│  │ Analyzer │ │ Planner  │ │Generator │ │ Generator│ │
│  │  Agent   │ │  Agent   │ │  Agent   │ │  Agent   │ │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────────┐  │
│  │ Executor │ │  Defect  │ │  Knowledge Manager   │  │
│  │  Agent   │ │ Analyzer │ │     Agent (RAG)      │  │
│  └──────────┘ └──────────┘ └──────────────────────┘  │
└──────────────────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────────┐
│           Draft → Review → Confirm           │
│  所有AI生成内容先存为 Draft，人工确认后生效    │
└──────────────────────────────────────────────┘
```

### 5.2 6大 Agent

| Agent | 角色 | 核心职责 | 工具 |
|-------|------|---------|------|
| RequirementAnalyzer | 需求分析师 | 分析需求文档、提取测试要点 | read_tool, analyze_tool |
| TestPlanner | 测试规划师 | 制定测试策略、生成方案 | plan_tool, suggest_tool |
| TestGenerator | 测试生成师 | 编写测试用例、覆盖场景 | generate_tool, review_tool |
| Executor | 测试执行师 | 驱动执行、解析结果 | execute_tool, parse_tool |
| DefectAnalyzer | 缺陷分析师 | 分析缺陷、定位根因 | analyze_tool, classify_tool |
| ReportGenerator | 报告分析师 | 生成质量报告、趋势分析 | report_tool, chart_tool |

### 5.3 Human-in-the-Loop 流程

```
AI Agent 生成内容
      │
      ▼
  ┌─────────┐
  │  Draft  │ ← 存为草稿，标记 ai_generated=true
  └────┬────┘
       │
  ┌────▼────┐     ┌──────────┐
  │ 人工审核  │────│ 退回修改  │
  └────┬────┘     └──────────┘
       │ 通过
  ┌────▼────┐
  │Approved │ ← 确认后入库，记录审核人
  └─────────┘
```

## 十、AI 工作流（Draft → Confirm）

### 10.1 AI测试用例生成流程
```
用户点击"AI生成"
  → TestGenerator Agent 开始工作
     → Planner: 分析测试方案，规划测试场景
     → Generator: 逐场景生成测试用例
     → Reviewer: 审查用例质量
  → 生成结果存为 ai_drafts (status=draft)
  → 前端展示 🤖 标记的草稿用例
  → 用户审核、编辑、补充
  → 用户点击"确认" → status=approved → 入库
  → 记录操作日志
```

### 10.2 AI需求分析流程
```
用户上传/创建需求
  → 点击"AI分析"
  → RequirementAnalyzer Agent
     → 分析需求内容
     → 提取关键测试点
     → 生成测试建议
  → 结果暂存 (ai_analysis 字段)
  → 人工审核分析结果
  → 确认/修改后批准
```

### 11.2 前端规范

- TanStack Query 管理服务端状态
- Zustand 管理客户端状态（auth, theme）
- 所有操作请求有 loading/error/success 状态
