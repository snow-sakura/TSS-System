## 快速入门

### 登录系统

1. 访问 http://localhost:5001
2. 输入用户名: `admin`
3. 输入密码: `admin123`
4. 点击"登录"

### 系统概览

登录后进入仪表盘，可以看到：
- 统计卡片：需求总数、测试点、测试方案、测试用例数量
- 测试流程图：显示各阶段的进度
- 快速操作：常用功能快捷入口
- 系统状态：当前测试执行状态

## 需求分析

### 创建需求

1. 点击左侧菜单"需求分析"
2. 点击"新建需求"按钮
3. 填写需求标题和内容
4. 点击"保存"

### AI分析需求

1. 在需求列表中找到要分析的需求
2. 点击"AI分析"按钮
3. 等待AI分析完成（约2-3秒）
4. 查看分析结果：
   - 摘要：需求的简要概述
   - 测试点：AI提取的测试点列表
   - 风险点：可能存在的风险
   - 建议：测试建议

### 管理需求

- **批准需求**: 点击"批准"按钮，需求状态变为"已批准"
- **拒绝需求**: 点击"拒绝"按钮，需求状态变为"已拒绝"
- **编辑需求**: 点击"编辑"按钮，修改需求内容
- **删除需求**: 点击"删除"按钮，删除需求

## 测试点管理

### 查看测试点

1. 点击左侧菜单"测试点管理"
2. 查看所有测试点列表
3. 使用筛选功能按类型、优先级、状态筛选

### 创建测试点

1. 点击"新建测试点"按钮
2. 填写测试点信息：
   - 标题：测试点的名称
   - 描述：测试点的详细描述
   - 类型：功能测试/接口测试/性能测试/安全测试
   - 优先级：高/中/低
   - 分类：所属模块
3. 点击"保存"

### 从需求创建测试点

1. 在需求分析页面，点击"查看分析"
2. 查看AI提取的测试点
3. 点击"创建测试点"按钮
4. 测试点自动创建并关联到需求

## 测试用例

### 创建测试用例

1. 点击左侧菜单"测试用例"
2. 点击"新建用例"按钮
3. 填写用例信息：
   - 标题：用例的名称
   - 前置条件：执行前需要满足的条件
   - 测试步骤：详细的操作步骤
   - 预期结果：期望的输出结果
   - 类型：功能测试/接口测试/性能测试/安全测试
   - 优先级：高/中/低
4. 点击"保存"

### AI生成用例

1. 在用例列表页面
2. 点击"AI生成"按钮
3. 选择关联的测试方案（可选）
4. 选择生成数量
5. 点击"开始生成"
6. 查看生成的用例列表

### 管理用例

- **查看详情**: 点击用例标题查看详细信息
- **编辑用例**: 点击"编辑"按钮修改用例
- **删除用例**: 点击"删除"按钮删除用例
- **批量选择**: 勾选多个用例进行批量操作

## 测试执行

### 执行测试

1. 点击左侧菜单"测试执行"
2. 在用例列表中勾选要执行的用例
3. 或点击"全选"选择所有用例
4. 点击"开始执行"按钮
5. 实时查看执行进度：
   - 进度条显示完成百分比
   - 当前阶段显示
   - 耗时统计

### 查看结果

1. 执行完成后自动显示结果
2. 结果统计包括：
   - 总用例数
   - 通过数
   - 失败数
   - 通过率

### 执行历史

1. 页面下方显示执行历史列表
2. 点击"查看"按钮查看详细结果
3. 包含每次执行的详细信息

## Web自动化测试

### 创建自动化脚本

1. 点击左侧菜单"Web自动化"
2. 点击"新建脚本"按钮
3. 填写脚本信息：
   - 脚本名称
   - 目标URL
   - 描述
4. 添加测试步骤：
   - 选择操作类型（导航/点击/输入/等待/断言）
   - 填写目标选择器
   - 填写值（如需要）
5. 点击"保存"

### 执行脚本

1. 在脚本列表中找到要执行的脚本
2. 点击"执行"按钮（绿色播放图标）
3. 确认执行
4. 等待执行完成
5. 查看执行结果

### 录制脚本

1. 点击"录制脚本"按钮
2. 输入目标网站URL
3. 点击"开始录制"
4. 在打开的浏览器中操作
5. 操作完成后自动保存脚本

## 系统配置

### LLM配置

1. 点击左侧菜单"大模型配置"
2. 点击"添加提供商"按钮
3. 选择提供商类型（OpenAI/Anthropic）
4. 填写：
   - 提供商名称
   - API密钥
   - Base URL（可选）
5. 点击"保存"
6. 点击"测试连接"验证配置

### 用户管理

1. 点击左侧菜单"用户管理"
2. 查看用户列表
3. 点击"新建用户"创建新用户
4. 填写用户信息：
   - 用户名
   - 邮箱
   - 密码
   - 角色（管理员/普通用户）
5. 点击"保存"

### 系统设置

1. 点击左侧菜单"系统设置"
2. 配置系统参数：
   - 通用设置：应用名称、基础URL等
   - LLM设置：默认模型、Temperature等
   - 测试设置：超时时间、重试次数等

## 常见问题

### Q: AI分析失败怎么办？

A: AI分析可能因为以下原因失败：
1. 未配置LLM提供商 → 在"大模型配置"中添加
2. API密钥无效 → 检查密钥是否正确
3. 网络问题 → 检查网络连接

系统会自动降级使用模拟数据，确保功能正常运行。

### Q: 测试执行卡住了怎么办？

A: 测试执行可能因为以下原因卡住：
1. 目标服务器不可用 → 检查服务器状态
2. 测试用例有错误 → 检查用例步骤
3. 网络超时 → 检查网络连接

可以在执行历史中查看详细的错误信息。

### Q: 如何备份数据？

A: 数据存储在SQLite数据库中，文件位于：
```
tts-paper-test/web/data/tss_platform.db
```
备份此文件即可。

### Q: 如何重置系统？

A: 删除数据库文件后重启服务：
```bash
rm tts-paper-test/web/data/tss_platform.db
python main.py web
```
系统会自动创建新的数据库和默认管理员用户。
